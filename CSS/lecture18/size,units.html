Physical Pixels Vs Logical Pixels

Physical Pixels

The actual tiny dots (RGB subpixels) that make up a screen.
Fixed per device; for example, a 1080p display has 1920×1080 physical pixels.
Higher-resolution screens (like Retina displays) have more physical pixels packed into the same screen size.

Logical Pixels (CSS Pixels, Density-Independent Pixels)

A unit of measurement used in web and UI design to maintain consistent sizing across devices.
Browsers and operating systems scale logical pixels based on the screen's pixel density (DPI/PPI).
On high-DPI screens, multiple physical pixels represent one logical pixel to ensure sharpness.
Example: A 100px wide element in CSS may occupy 200 or 300 physical pixels on high-resolution displays.

Formula for Scaling:

Logical Pixels= Physical Pixels/Device Pixel Ratio(DPR);

PPI (Pixels Per Inch) vs. GPI (Grid Per Inch)

Definition: PPI refers to the number of physical pixels displayed per inch on a screen.
Usage: Primarily used for digital screens (monitors, smartphones, tablets).
Impact: Higher PPI results in sharper and more detailed images.
Example:
A 1080p display (1920×1080) on a 24-inch monitor has around 92 PPI.
A Retina display on an iPhone may have ~326 PPI or more for crisper visuals.

Definition: GPI refers to the number of grid cells per inch, mainly used in graphical representations, printing, and scanning.
Usage: Found in grid-based systems, old-school bitmap displays, and some scanning technologies.
Impact: Higher GPI improves the accuracy of printed graphics, CAD drawings, or scanning resolution.
Example:
A printed circuit board (PCB) might be designed with 10 GPI to define grid spacing for precise component placement.
Older CRT monitors sometimes referenced GPI to describe scan resolution.
